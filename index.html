I challenge my students as follows:

  • Everyone else seeks solutions that are more intricate and complex;
  • But we know that empirically and theoretically, that the world is a simpler place than we might have believed;
  • So can you do "it" better, with less?.
The results are startling; for example, see the sample of "less, but better" results, below (for more examples, see all my other papers ).

Defect prediction

2007

Surprisingly effective defect predictors can be built from simple static code attribiutes.


2010

Static code defect predictors have inherent limtations, which can be fixed via new learner that better understand the business goals.


Search-based SE

2002

Search-based SE methods can critically assess long held SE truisms.


2013

Active learners can reduce the cost of search-based SE by orders of magnitude.


2016

Very simple optimizers can dramatically improve the performance of data miners learning software quality predictors.


Transfer learning

2013

Clustering tools enable transfering lessons learned between software projects.


2016

Ultra-simple transfer learning methods (called "bellweathers") enable effective transfer of lessons learned.


Requirements Engineering

2002

Contrast set learners find simple controllers in requirements models.


2003

Contrast set learners can explain enormous decision trees (6000 node) leared from complex requirements models just 6 rules.


Privitization

Feature and instance selectors let software projects share and reason about privitized data.


Active learning

2013

Active learners can estimate large software projects after just a few samples.


Text mining

Social metrics

Effort estimation

Expert systems
